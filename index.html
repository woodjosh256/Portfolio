About

Four years ago, I started coding small games on my TI-84 graphing calculator. I quickly developed a passion for programming and loved how one could create nearly anything just by writing lines of code. I soon moved on to other more capable languages, primarily Python and Java. I began to spend a lot of my free time on small programming projects and even co-founded my high school’s coding club. I’ve decided that I want to turn my passion for programming into my career. I am now studying software engineering at the University of Wisconsin – Platteville.

Outside of programming, I like to spend my time mountain biking, hiking/backpacking, and listening to music. A long-term goal of mine is to thru-hike hike the Appalachian Trail.

Whole Milk Calculator

Summer 2019

I worked with technical and marketing experts at Milk Products Inc (part of Land O’Lakes) to develop an easy-to-use app that helps calf raisers effectively feed their calves. The app is programmed using HTML/CSS/JavaScript with Adobe PhoneGap. Some of the challenges for this project included finding a way to write the code that budget/time constraints as well as finding a way for the app to support two different languages. Programming the Whole Milk Calculator app was my first paid programming project as well as my first project with HTML/CSS/JavaScript. The app will be available on Google Play and the App Store within the next few weeks.

The Journey of Bella Shores

Spring 2019

This was a game that I developed for a game jam run by Repl.it (a game jam is a competition in which one develops a game that is related to a theme within a short period of time). Over about a month I created the game’s art, wrote the game’s story, and programmed it in Python (using Pygame for graphics). The most difficult parts of this project involved creating an engine that could manage each boat’s physics and collisions, making a system for displaying the UI elements, and figuring out how to randomly generate the map. If I were to do this project again, I would try to have fewer features and focus on making those features more thought out. I felt that in the end, each element of the game wasn’t very much fun because I had focused on adding as much as I could instead of taking time to plan each feature out well. Poe - The Autonomous Poet

Febuary 2019

This is a smaller project that I completed for an AI themed code jam run by Repl.it (a code jam is a competition where one designs a program about a theme within a short period of time). The program reads a large text (usually a long book) to find which words commonly follow other words. It then attempts to use what it learned from the texts it read and the Datamuse API to make a sonnet or haiku about a certain theme with a Markov Chain algorithm. The poems the program writes never make any sense but reading what it makes can illustrate the writing style of the texts it read. This program didn't place in the competition but was listed as an honorable mention. The code and program can viewed/tested below.
